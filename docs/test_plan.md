## テスト基盤の注意事項（バグではない）

- **試薬の DELETE API が存在しない**ため、テストで作成した `TEST_アセトン` が残り続ける。  
  2回目以降の試薬系テスト（T08/T09/T10）は `already exists` エラーで FAIL する。  
  → 試薬 DELETE API を追加するか、テスト前にDBをリセットすることで解消できる。

- **DB に既存の ARRIVED アイテムがある場合**、T06「確認したボタン→ブロック消滅」は PASS しない。  
  → テスト用 DB を用意するか、テスト開始前に全アイテムを NONE にリセットすることで解消できる。
